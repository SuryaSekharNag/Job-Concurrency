🌙 HTML CI Pipeline using GitHub Actions (Concurrency)

📁 Step 1 — Project Structure

your-project/
│
├── index.html
└── .github/workflows/ci.yml

⚙️ Step 2 — Workflow File Location

Create file:

.github/workflows/ci.yml

🧠 Step 3 — YAML Workflow Content

name: HTML CI (Concurrency Standard)

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Validate project
        run: echo "HTML project pushed successfully"

      - name: List files
        run: ls -la

🚀 Step 4 — Push to GitHub

git init
git add .
git commit -m "HTML CI concurrency project"
git branch -M main
git remote add origin https://github.com/SuryaSekharNag/Job-Concurrency.git
git push -u origin main

🎯 Concept Explained

Concurrency ensures only the latest workflow runs and cancels previous builds, preventing duplicate CI execution and saving CI resources.

✅ Outcome

  • Project pushed to GitHub
  • Workflow runs automatically
  • Concurrency optimization enabled
